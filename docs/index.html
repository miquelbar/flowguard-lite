Overview

FlowGuard Lite is a lightweight NDR (Network Detection and Response) product designed for homelabs, small offices, prosumer networks, and teams who do not have a full security operations center (SOC).

It acts as a lightweight flow collector and correlates router/firewall telemetry with optional Suricata IDS events. By building device-centric behavioral baselines, FlowGuard Lite detects volumetric attacks, DDoS floods, and anomalies, presenting them with clear, human-readable explanations.

Lightweight Performance

Quick Setup

FlowGuard Lite can be deployed either natively or using Docker Compose.

Docker Compose Deployment (Recommended)

Create a docker-compose.yml file:

version: '3.8'

services:
  flowguard:
    image: flowguard:latest
    container_name: flowguard
    restart: unless-stopped
    ports:
      - "8080:8080"        # Web UI Dashboard & REST API
      - "2055:2055/udp"    # NetFlow / IPFIX port
      - "6343:6343/udp"    # sFlow port
    volumes:
      - flowguard_data:/data
    environment:
      - FLOWGUARD_LOG_LEVEL=info

volumes:
  flowguard_data:

Start the container:

docker compose up -d

Host-Native Run

To run natively on your host machine, compile the Go binary and run it:

make build
./bin/flowguard -config /path/to/config.yaml

Exporter Configuration

To start monitoring your network, configure your routers or firewalls to export flow logs (NetFlow v5/v9, IPFIX, or sFlow) to the IP address of your FlowGuard Lite host.

Note: NetFlow/IPFIX normally exports on UDP port 2055, and sFlow exports on UDP port 6343.

Ubiquiti UniFi Gateway Config

In your UniFi Controller settings:

  1. Navigate to Settings > System > Advanced.
  2. Scroll down to the NetFlow section.
  3. Enable NetFlow, and set the destination IP to your FlowGuard Lite container host.
  4. Set the port to 2055.

Architecture

FlowGuard Lite operates with a clean decoupled design focusing on efficiency and storage boundaries:


   +---------------------------------------------+
   |             Router / Firewall               |
   +----------------------+----------------------+
                          | (IPFIX / sFlow)
                          v (UDP)
   +----------------------+----------------------+
   |        Flow Collector (Worker Pool)         |
   +----------------------+----------------------+
                          | (Normalized Event)
                          v
   +----------------------+----------------------+
   |             Memory Aggregator               |
   +----------------------+----------------------+
                          | (Batch writes - 15s)
                          v
   +----------------------+----------------------+
   |        Repository Interface Driver          |
   +----------------------+----------------------+
                          |
                          v
   +----------------------+----------------------+
   |      Daily SQLite Evidence Shards           |
   +---------------------------------------------+

Using a decoupled Repository pattern, the core business logic (aggregations, baseline modeling, and alert evaluation) remains completely database-agnostic. While SQLite daily shards are the default storage for homelabs, DuckDB can be plugged in for heavy analytic query workloads by writing a new implementation of the database interface.

API Reference

FlowGuard Lite exposes a clean, JSON-based REST API for dashboard telemetry and third-party integrations.

System Endpoints

  • GET /api/health: Basic daemon health status and worker statistics.
  • GET /api/exporters: List of all active flow exporters with last-seen counters.

Analytics Endpoints

  • GET /api/top/sources: Top local speaking host IPs.
  • GET /api/top/destinations: Top destination IPs.
  • GET /api/top/ports: Top ports/protocols.
